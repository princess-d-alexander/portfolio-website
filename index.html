About This Portfolio

This portfolio documents my work and competencies developed throughout the CSC-324 software development course. The pages in this portfolio demonstrate my contributions to collaborative game development, interface design, software architecture, testing infrastructure, usability evaluation, and engineering workflow practices.

The portfolio includes examples of code, interface revisions, testing systems, design artifacts, collaborative development processes, and reflections on the technical and human-centered decisions involved in building the project.

Project Title / Type

PaperQuest — Idle / Incremental Game built using Lua and the Love2D framework.

PaperQuest is a resource-management idle game centered around collecting wood, producing paper, unlocking progression systems, and managing automated gameplay mechanics such as factories, meeples, prestige systems, and conditional dialogue.

Reflection

My main role on the project focused on UI design, dialogue systems, usability revisions, and testing infrastructure. I worked heavily on the visual redesign of the game during the meeple integration sprint and helped organize gameplay systems into clearer and more modular structures.

Through this project I became more comfortable working with GitHub, collaborative software development, Love2D, and updating game state programming. I also learned how important abstraction, testing, and interface clarity are when building larger software projects with a team.
